## Что изменилось

### 1. База данных (бот + сайт)
- Новая таблица **`servers`** — хранит все x-ui серверы (URL, логин, пароль, сертификат, локация, вес, статус)
- Таблица **`keys`** теперь связана с сервером через `server_id`
- Автомиграция: при старте добавляет `server_id` к существующим ключам и создаёт дефолтный сервер из `XUI_URL` (если БД пуста)

### 2. Ядро бота
- **`XuiClient`** — убран singleton, теперь создаётся для конкретного сервера
- **`XuiClientFactory`** — фабрика клиентов с кэшем по `server_id`, проверкой доступности и алгоритмом выбора наименее загруженного сервера
- **`ServerDao`** / **`Server`** — модель и доступ к данным серверов

### 3. Telegram-бот
- **Покупка ключа** — сначала выбор сервера, потом inbound, потом тариф. Автоматически выбирается лучший сервер при покупке через магазин/тарифы
- **Админ-команды**:
  - `/servers` — список серверов со статусом доступности
  - `/add_server` — FSM для добавления сервера
  - `/del_server` — удаление сервера
  - `/xui_inbounds` — просмотр inbound'ов с выбором сервера
  - `/add_key` — добавление ключа пользователю с выбором сервера
- **Профиль пользователя** — в деталях ключа отображается имя сервера

### 4. Сайт (Spring Boot)
- **`Server`** entity + `ServerRepository`
- **Админ-панель** — новый раздел "Серверы" с таблицей и формой добавления
- **Профиль** — для каждого ключа показывается название сервера

### 5. Обратная совместимость
- `XUI_URL` в `.env` стал **опциональным**
- Существующие ключи автоматически привязываются к дефолтному серверу
- Все тесты проходят (12/12)

### Сборка
```bash
# Бот
mvn clean package
# Сайт
cd site && mvn clean package
```
